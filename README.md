# lxns_b50

lxns_b50 是开发组内部维护的 NoneBot2 舞萌 DX 查分与分析插件，集成落雪 (LXNS) 与水鱼 (Diving-Fish) 双数据源，用于成绩查询、战力分析、名片生成、课题挑战和群内小游戏等场景。

本仓库当前不作为公开通用插件发布。README 主要记录项目定位、功能边界和维护注意事项，不提供完整部署流程、公开配置说明或外部调用方式。

## 当前状态

- 维护状态：开发组内部维护中。
- 使用范围：面向现有机器人环境、既有数据源和固定群聊场景，不保证外部环境可直接运行。
- 数据源边界：功能依赖 LXNS、Diving-Fish、曲库资源和机器人消息适配层的可用状态。
- 许可证状态：暂未选择开源许可证。未经明确许可，不建议再分发、商用或用于生产部署。

## 功能介绍

### 核心查分

- `b50` / `B50`：生成 Best 50 个人综合成绩面板。
- `ap50` / `AP50`：生成 AP 50 纯收曲全成就大图。
- `minfo <曲名/ID>`：查询单曲在当前玩家处的游玩详情与达成率历史。
- `ginfo <难度><曲名/ID>`：渲染全服此谱面达成率与 FC 比例统计图表。
- `分数线 <难度><曲名/ID> <达成率目标>`：计算达成目标率所需的 Perfect/Great/Good/Miss 容错值。

### 玩家个性化

- `dxpass`：渐进式名片生成助手。
- `mai曲线`：绘制 Rating 历史演变折线走势。
- `mai状态` / `mai个人中心`：诊断双平台绑定状态与数据同步情况。
- `切换数据源 <水鱼/落雪>`：切换玩家查分默认输出源。
- `mai名片` / `mai信息`：生成玩家个人属性名片大图。
- `mai排行` / `战力排行`：渲染全服 TOP 20 战力与装扮大盘。
- `mai分析` / `战力分析`：分析 Best 50 成绩占比与分布情况。
- `mai速报` / `mai打卡`：提取最近游玩的优秀成绩生成打卡图片。
- `mai底力` / `底力`：高难成绩评级与积分大图。

### 曲目检索与进度

- `查歌 <关键词>`：模糊检索曲目名。
- `id <曲目ID>`：调取谱面底标参数与难度详情。
- `定数查歌 <定数>`：查询对应定数范围的歌曲。
- `bpm查歌 <BPM>`：查询指定 BPM 的歌曲。
- `<版本><目标>完成表`：查看特定版本的牌子达成视觉化进度。
- `<版本><目标>进度` / `<等级><评价>进度`：查看达成目标的完成曲数比例与待完成曲目清单。
- `我要上<分数>`：根据玩家 Best 50 推荐上分曲目。

### 互动小游戏

- `猜歌`：开启群内趣味歌词或信息猜曲游戏。
- `猜曲绘`：开启看局部曲绘大图猜歌。
- `听歌猜歌`：提取歌曲音频片段在群内播放以猜歌。
- `别名猜歌`：使用梗别名作为谜题猜官方中文曲名。

## 维护注意事项

- 与 LXNS、Diving-Fish 和曲库资源相关的异常应明确区分，避免把外部数据源波动误判为插件逻辑错误。
- 涉及绑定状态、成绩缓存、曲库同步和图片生成的改动，应先在开发组环境验证。
- 图片渲染逻辑应注意字体、缓存、资源缺失和长文本溢出问题。
- 游戏和互动功能应控制状态生命周期，避免旧状态影响新一轮交互。
- 不要把真实 token、Cookie、数据库凭据或运行时数据提交到仓库。
